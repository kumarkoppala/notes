symlink
========
Chrome -> ProgramFiles/google-chrome
Desktop > Google Chrome -> ProgramFiles/google-chrome

1. inode of symlink and actual file is different
2. if we delete actual file, symlink will break
3. uses for backward compatability, version switching and then rollback
4. you can create symlink directories also

hardlink
========
1. same inode for hardlink
2. even we delete the file, other link files still exist, we can access
3. used for backups
4. actual content will not be deleted until all the links are deleted
5. hard link not allowed for directory
